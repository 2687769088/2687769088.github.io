# 面向高性能计算及新型计算的系统结构的探索

## 研究一：GPU 处理器

- 构建模拟平台，改进计算访存结构
  - Cache 和 memory 优化管理，数据一致性
  - 基于访存特征（在模拟器中收集）进行优化
- GPU 程序性能分析
  - rocprof 开发（依赖硬件实现，只能跑在 AMD GPU）
    - 对标 nvprov
  - 指令集精细力度分析
    - 1k 次指令收集信息
- GPU 前沿展望

- memory 和 Cache 之间又加了一层 SADRAM

- 芯片面片省掉三分之一，没有 shader 等图像渲染单元

- GPU 生态做软件的比做硬件的多

- GPU 中间指令-》实际型号

## 研究二：存储

- DRAM 受制于数据恢复操作
  - 性能和良品率
- NVM 容量高访问速度慢
  - 读写延迟高，性能慢
- 传统存储：加快数据恢复操作
  - 关联刷新以部分恢复，重组构建快速区域
  - 结合应用容错性，近似计算加快恢复
- 新型存储：缩短数据存取时延
- 改进存储单元结构和编码数据
- 应用于 GPU 处理器，解决潜在性能问题

  - 容错，程序中指定关键数据和非关键数据
  - 调整 request，给关键数据提升权

- AMD 当前两层 Cache，下一代四层 Cache
